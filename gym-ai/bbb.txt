SMART GYM APPLICATION — PFE COMPLET
Vue d'ensemble du projet
Une application complète de gestion de gym intelligente avec prédictions IA pour détecter la fatigue et le risque de blessure des athlètes. L'application cible à la fois les membres (sportifs) et les coachs, disponible en mobile (Flutter) et web.

ARCHITECTURE GLOBALE
Flutter (Mobile + Web)
        ↓
Spring Boot (Backend - Java)
        ↓
    PostgreSQL (Base de données)
        ↓
FastAPI (Service IA - Python)

PHASE 1 : MODULE IA (PYTHON + FASTAPI) — 100% TERMINÉ
Datasets utilisés

Fatigue : Exercise and Fitness Metrics Dataset (3864 lignes)
Blessure : Collegiate Athlete Injury Dataset (200 lignes)

Modèles entraînés

Prédiction Fatigue : Random Forest

Accuracy : 0.63
Features : duration, bmi, age, gender
Output : {"fatigue": 0/1, "label": "normal/fatigué", "confidence": 0.99}


Prédiction Blessure : Random Forest + SMOTE (pour gérer le déséquilibre)

Accuracy : 0.98
Features : age, training_intensity, training_hours_per_week, recovery_days_per_week, fatigue_score, load_balance_score, acl_risk_score
Output : {"injury_risk": 0/1, "label": "risque faible/élevé", "confidence": 0.79}



Fichiers Python

train.py : Entraîne les deux modèles avec SMOTE
app.py : API FastAPI avec endpoints /predict_fatigue et /predict_injury
test_api.py : Tests des endpoints

Démarrage
bashpython train.py
uvicorn app:app --reload
API disponible sur http://127.0.0.1:8000

FLUX COMPLET D'UNE SÉANCE

Utilisateur Flutter → Remplit formulaire (durée, intensité, poids)
Flutter → POST /sessions/member/{memberId} à Spring Boot
Spring Boot → Crée TrainingSession dans PostgreSQL
Spring Boot → GET /api/ai/predict/{memberId}/{sessionId}
Spring Boot → POST à FastAPI /predict_fatigue + /predict_injury
FastAPI → Calcule BMI, formatte features, prédit
FastAPI → Retourne {"fatigue": 1, "label": "fatigué", "confidence": 0.99}
Spring Boot → Retourne réponse à Flutter
Flutter → Affiche résultats dans NewSessionScreen
